# 第 05 步：超参数调优与多模型集成学习指南 (Guide)

本指南针对 `05_hyperparameter_tuning_and_ensemble.ipynb` 笔记本中的操作流程与实现原理进行详细说明，该阶段是比赛打榜、提分最为关键的一步。

## 🎯 核心目标
1. 针对 `04_model_training.ipynb` 中跑通的 9 大基线生存分析模型，使用 **Optuna** 进行超参数盲搜调优。
2. 针对本次比赛高度关注 **“48小时精度”** 的特点，定制化设定评估 Loss （即 $0.7 \times  Brier_{48h} + 0.3 \times (1 - C_{index})$ ），牵引模型专门对 48h 时刻的输出概率进行刻意标定。
3. 借助集成学习法（Ensemble）平权归一融合所有调优后模型的预测输出，以获取兼顾局部准确（树群）和全局鲁棒（线性基线）的“最强答卷”。

---

## 🛠️ 第一阶段： Optuna 并发超参数搜索

`05` 笔记本中实现了基于 `scikit-survival` 规范的统一接口代理及评估封装。在这个阶段，我们要对如下 9 种存活分析模型逐一摸彩：

### 涵盖评估的模型矩阵：
- **树分集组合（强局部）**：`RandomSurvivalForest`, `GradientBoostingSurvivalAnalysis`, `ComponentwiseGradientBoostingSurvivalAnalysis`, `ExtraSurvivalTrees`, `SurvivalTree`
- **线性降维/边际打分（强全局）**：`CoxnetSurvivalAnalysis`(充当 MLP的惩罚替代), `CoxPHSurvivalAnalysis`, `FastSurvivalSVM` (支持向量机生存预测)
- **原生生存提升**：`NativeXGBoostSurvival` `(survival:cox)` 

### 调优的工作流拆解：
1. **输入装配**：对于部分要求数据完全不能有共线性或绝对量纲差距的线性模型（如 SVM 与 Cox 家族），代码内部自动切换拉取 `X_train_scaled` （即 `StandardScaler` 处理后的入参）。对于原生 XGBoost 生存回归，自动切换组装符合 `hist` 计算特性的标签矩阵。
2. **K-Fold 交叉验证打分：** 在内部实施 3 折交叉验证分割 `X_train`。
3. **搜包界限：** 根据 `optuna.suggest_int / float` 进行盲投，探索每种模型的最佳参数（如 XGB 的学习率 `learning_rate` 搜索区设定为 `0.01~0.2` 等）。
4. **生成最佳词典：** Optuna 根据打分配比，最终留存的参数会被全量收入在 `best_params_dict` 字典变量中。

> ⚠️ **非常重要**：当前代码块为演示流程，`n_trials=3`（仅各自碰运气 3 次）。真正运行大阶段打榜时，请修改第 3 个单元格循环里的数值为： `study.optimize(..., n_trials=30)`，让 GPU 熬一会夜。

## 🧬 第二阶段：全量二次复飞与模型对比 (Retraining)

搜索获得了 `best_params_dict` 参数配方表后，这阶段原先的 3 折交叉切分会被抛弃不用。我们将直接带着这套最优参数去**重读覆盖整个 100% 测试集**。

在最新的代码中，我们加入了**调参前后对比机制**：
所有的模型都会分别用**默认参数（Baseline）**和**最佳参数（Tuned）**对全量数据学习一次。在控制台上会直观打印调参前后 C-Index 得分的提升差值，这样能快速暴露出某些树模型会不会出现“搜出奇异过拟合数值”的情况。

训练结束后，调优模型会阅读未知的测试试卷，存下它内心推测出的`test_risk_scores` (综合危险值矩阵) 用于最后大阶段的整合。

## 🔗 第三阶段：多重策略集成学习 (Ensemble Assembly)

由于 9 款生存分析模型彼此算法截然不同，它们输出 Risk Score 的**绝对值刻度可以说是天差地别**（有的是 -5 到 10，有的是几十万的累计指数）。这带来了一个问题：绝对数字失去了横向硬比较的意义，在比赛中单纯关注数值是不合理的，关键要看**排序关系（C-Index）**。

因此我们在最终结果导出环节布置了三种高阶多模型平权融合手段，全面榨起分数：

1. **算术均值集成 (Mean)** 
   使用 `MinMaxScaler` 强行把各个“不同脾气”模型的打分区间全都规整到 `0.0~1.0` 之间后，取无脑大平均。虽然中规中矩，但它能消除大部分因刻度导致的融合失败。

2. **中位数集成 (Median)** 
   由于个别树模型在面临长尾未知样本时会“发疯”给出一个极其极端的概率值，简单求均值会被干扰拉偏。取所有模型打出的压缩分的中位数，可以极大屏蔽这类单个模型的疯狂噪音，提供最平稳的兜底指标。

3. **排秩百分位集成 (Rank Averaging) 🌟【冲榜最强招数】**
   这一步**不看成绩数值，只看名次排位**！我们把每个样本在这个模型中打出的危险度转换为其所在的百分位名次（比如风险排在后 90% 的人记为 0.9），这种方法直接暴力对准了 C-Index “只惩罚排错位”的规则考核属性，通常会在最终 Kaggle 评分中拿到奇效。

## 📄 产出 CSV 文件的含义解读

代码终结后会于文件夹输出三份不同的答案纸格式（均可无缝用于官网打榜提交）：
- `submission_ensemble_mean.csv` 
- `submission_ensemble_median.csv`
- `submission_ensemble_rank.csv`

每份提交表格中的结构与内部物理含义如下：

| 列名 | 含义与解析 |
|---|---|
| **`event_id`** | 测试集中每条数据的唯一事件/设备 ID。比赛组委就是依赖这列去数据库对照真实生存情况打分的。 |
| **`prediction`** | 模型推演出来的 **风险指示相对偏置 (Risk Score)**。 |

**分数如何理解？**
- 此值并不是绝对时间的概率预测！它是一个**倒挂**的综合危险相对值。
- **分数越高**，模型越认为这个单元“快遇到火灾/出事燃烧”的时间会更早、风险极高。
- **分数越低**，说明模型认定它大概率会活得时间非常漫长、安然无恙。

将这三份文件分别按情况提交上传至打榜系统，看一看三种集成加权方法哪个的 C-Index 指标表现最为优秀，然后就可以在 Final 报告中拍板定下您的“最优王者策略”了！
